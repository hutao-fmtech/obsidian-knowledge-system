# 📎 附件库

> 统一管理笔记中使用的图片、文件等附件

---

## 💡 什么是附件库？

附件库用于存放笔记中引用的各类文件：
- 🖼️ 图片（截图、照片、图表）
- 📄 PDF文档
- 🎵 音频文件
- 📊 数据文件（Excel、CSV等）
- 📦 其他文件

### 为什么需要附件库？

- ✅ **集中管理**：所有附件统一存放
- ✅ **便于维护**：清理和备份更方便
- ✅ **避免混乱**：不会散落在各个文件夹
- ✅ **跨笔记引用**：同一文件可被多个笔记使用

## 📝 使用规范

### 文件命名规范

#### 基本原则
- 使用描述性名称
- 避免特殊字符
- 使用连字符或下划线
- 添加日期前缀（可选）

#### 命名格式

**图片**：
```
描述-YYYYMMDD.png
项目名-功能-截图.png
```

**示例**：
- `技术架构图-20260122.png`
- `用户界面-登录页面-截图.png`
- `数据流程图.png`

**文档**：
```
文档类型-主题-版本.pdf
项目名-文档名-YYYYMMDD.pdf
```

**示例**：
- `技术方案-系统重构-v1.0.pdf`
- `产品手册-用户指南-20260122.pdf`

**音频**：
```
类型-主题-日期.mp3
会议录音-YYYYMMDD.mp3
```

**示例**：
- `会议录音-技术评审-20260122.mp3`
- `播客-知识管理方法.mp3`

## 🖼️ 图片管理

### 支持的格式
- PNG（推荐用于截图和图表）
- JPG/JPEG（推荐用于照片）
- GIF（动图）
- SVG（矢量图）
- WebP

### 图片优化建议

#### 截图
- 使用 PNG 格式
- 适当裁剪，去除无关区域
- 必要时添加标注

#### 照片
- 使用 JPG 格式
- 压缩过大的图片（建议 < 1MB）
- 保持合适的分辨率

#### 图表
- 优先使用 SVG（可缩放）
- 或使用高分辨率 PNG
- 保持清晰可读

### 在笔记中引用图片

```markdown
![[图片名称.png]]
![[图片名称.png|300]]          # 指定宽度
![[图片名称.png|300x200]]      # 指定宽高

# 使用Markdown语法
![描述文字](../90-附件/images/图片名称.png)
```

## 💡 最佳实践

### 1. 统一存放
所有附件都放在 `90-附件/` 下，不要散落在笔记文件夹中

### 2. 分类存储
按照文件类型分类存放（图片、文档、音频）

### 3. 描述性命名
文件名要能说明内容，方便查找

### 4. 适度压缩
- 图片保持适当质量和大小
- 避免存储超大文件
- 必要时使用外部存储

### 5. 定期清理
- 删除不再使用的附件
- 清理重复文件
- 归档历史附件

### 6. 版本控制
重要文件保留版本记录：
```
文档名-v1.0.pdf
文档名-v1.1.pdf
文档名-v2.0.pdf
```

## ⚠️ 注意事项

### 文件大小控制

**建议限制**：
- 单个图片 < 2MB
- 单个文档 < 10MB
- 单个音频 < 50MB
- 总附件库 < 5GB（根据需要调整）

**超大文件处理**：
- 使用外部云存储
- 在笔记中添加链接
- 定期清理和归档

### 敏感信息保护

**避免存储**：
- 密码和密钥
- 个人身份信息
- 银行账号信息
- 未加密的敏感文档

**如需存储**：
- 加密后再存储
- 使用密码保护
- 定期检查权限

### 版权注意

- 尊重图片版权
- 注明来源和引用
- 不公开分享受保护的内容
- 个人使用时注意授权
