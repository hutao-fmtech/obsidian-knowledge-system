# 🗂️ 索引卡片 (MOC)

> Map of Content：知识地图，连接相关主题的卡片

---

## 💡 什么是索引卡片？

索引卡片（MOC - Maps of Content）是**知识的导航地图**，将某个主题下的相关卡片组织起来。

### 特点
- ✅ 枢纽作用：连接相关知识
- ✅ 层级结构：从概览到细节
- ✅ 动态发展：随知识增长演进
- ✅ 多维视角：提供不同组织方式

### 与文件夹的区别
| 维度 | 文件夹 | MOC |
|------|--------|-----|
| 结构 | 树状，单一归属 | 网状，多重链接 |
| 灵活性 | 固定分类 | 灵活重组 |
| 视角 | 单一视角 | 多维视角 |
| 发现 | 需要记住位置 | 通过链接发现 |

---

## 📂 MOC的类型

### 主题MOC
围绕某个主题组织相关卡片

**示例**：
- MOC-知识管理方法论
- MOC-技术管理
- MOC-职业发展

### 项目MOC
为某个项目组织相关资源

**示例**：
- MOC-系统重构项目
- MOC-学习Rust
- MOC-写作计划

### 时间MOC
基于时间维度组织

**示例**：
- MOC-2026年度回顾
- MOC-第一季度OKR

## ✍️ 创建索引卡片

### 何时创建MOC？

#### 信号1：卡片过多
某个主题的卡片超过5-7个，需要组织

#### 信号2：发现联系
发现多个卡片之间有内在联系

#### 信号3：需要概览
想要快速了解某个主题的全貌

#### 信号4：准备输出
准备写文章或做分享，需要梳理思路

### 如何创建？

1. **使用模板**
   - [[00-系统/templates/moc-template|MOC模板]]

2. **命名规范**
   - 格式：`MOC-主题名称`
   - 示例：`MOC-知识管理方法论`

3. **结构设计**
   ```markdown
   # MOC-主题名称

   ## 概述
   主题的简要介绍

   ## 核心概念
   - [[概念1]]
   - [[概念2]]

   ## 方法论
   - [[方法1]]
   - [[方法2]]

   ## 个人观点
   - [[观点1]]
   - [[观点2]]

   ## 相关资源
   - [[资源1]]
   - [[资源2]]
   ```

4. **添加标签**
   - #索引卡片
   - #MOC
   - #主题标签

## 🏗️ MOC的构建方法

### 自底向上（Bottom-up）
从已有卡片中发现模式，归纳总结

**步骤**：
1. 收集相关卡片
2. 寻找共同主题
3. 组织成层级结构
4. 创建MOC索引

### 自顶向下（Top-down）
先设计框架，再填充内容

**步骤**：
1. 确定主题和范围
2. 设计知识框架
3. 识别缺失部分
4. 逐步补充内容

### 混合方式
结合两种方法，迭代优化

## 📐 MOC的组织维度

### 按概念层次
```
主题
├── 基础概念
├── 核心原理
├── 应用方法
└── 实践案例
```

### 按学习路径
```
学习主题
├── 入门必读
├── 进阶内容
├── 高级主题
└── 延伸阅读
```

### 按问题导向
```
问题主题
├── 问题是什么
├── 为什么重要
├── 如何解决
└── 案例分析
```

### 按时间维度
```
项目主题
├── 背景和目标
├── 进展记录
├── 经验教训
└── 成果总结
```

## 💡 MOC最佳实践

### 保持简洁
- 不要过度嵌套
- 突出重点内容
- 使用清晰标题

### 提供上下文
- 简要说明主题
- 解释组织逻辑
- 标注重要性

### 动态更新
- 添加新卡片
- 调整组织结构
- 删除过时链接

### 多重视角
- 同一主题可以有多个MOC
- 从不同角度组织
- 满足不同需求

## 🔗 MOC网络

### MOC间的关系
- **上级MOC**：更宏观的主题
- **下级MOC**：更具体的子主题
- **平行MOC**：相关的其他主题

### 构建知识网络
```
总MOC：个人知识体系
├── MOC-知识管理
│   ├── MOC-笔记方法
│   └── MOC-工具使用
├── MOC-技术管理
│   ├── MOC-团队管理
│   └── MOC-技术决策
└── MOC-职业发展
    ├── MOC-技能提升
    └── MOC-职业规划
```
