# PARA方法论

> PARA是一种基于可操作性的信息组织系统

## 💡 核心概念

**PARA** 是由 Tiago Forte 提出的信息组织方法，将所有信息按照可操作性分为四个层级：

### 四个层级

1. **Projects (项目)** - 有明确目标和截止日期的短期任务
2. **Areas (领域)** - 需要长期维护的责任领域
3. **Resources (资源)** - 未来可能有用的参考材料
4. **Archives (归档)** - 不再活跃的内容

## 🎯 我的理解

PARA 的核心价值在于==按照可操作性而非主题分类==。这解决了传统分类系统的一个关键问题：同一个信息可能属于多个主题，但它只有一个操作优先级。

### 关键区别

| 维度 | 项目 | 领域 | 资源 |
|------|------|------|------|
| 时间性 | 短期（有终点） | 长期（持续） | 不确定 |
| 目标 | 具体可达成 | 维持标准 | 备用参考 |
| 例子 | 完成季度OKR | 职业发展 | 技术文章收藏 |

## 💼 实践应用

在我的知识系统中的体现：

- **02-项目**: 当前正在推进的工作项目、学习计划
- **03-领域**: 职业发展、健康管理、个人成长等长期关注的方向
- **04-资源**: 阅读笔记、课程资料、参考文档
- **05-归档**: 已完成的项目、不再活跃的内容

### 使用原则

1. **收件箱优先** - 所有信息先进入 01-收件箱
2. **定期整理** - 每周将收件箱内容分配到 PARA 四层
3. **动态调整** - 项目完成后移入归档，新关注点创建领域
4. **最小化分类** - 不过度细分，保持简单

## 💭 延伸思考

**为什么 PARA 比主题分类更有效？**

主题分类的问题：一篇关于"Python机器学习"的文章，应该放在"编程"还是"机器学习"文件夹？这种模糊性导致犹豫和浪费时间。

PARA 的答案：问自己"我现在要用它做什么"？
- 如果是为了完成某个项目 → 02-项目/[项目名]
- 如果是长期学习的方向 → 03-领域/技术学习
- 如果只是参考材料 → 04-资源/技术文章

这种按操作优先级分类，让信息组织变得更加清晰和高效。
