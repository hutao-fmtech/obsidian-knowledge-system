## 方法概述

PARA分类决策树是一套快速判断笔记、文件或信息应该归入 Projects（项目）、Areas（领域）、Resources（资源）还是 Archives（归档）的决策流程。通过回答一系列简单问题，你可以在 10 秒内做出准确的分类决策。

核心理念：**通过可执行性判断。项目有明确终点，领域需要持续维护，资源是外部参考，归档是已完成内容。**

## 适用场景

- 创建新笔记时快速分类
- 清理收件箱时批量处理
- 整理现有笔记系统
- 项目完成后归档决策
- 领域与项目边界模糊时

## 具体步骤

### 步骤 1：理解 PARA 四个类别的本质

**Projects（项目）- 有终点的目标**
- 有明确的截止日期或时间范围
- 有可衡量的完成标准
- 完成后可以归档
- 示例：学习 Docker 容器化技术（3 个月学习计划）、减重 10 斤（6 个月健康目标）

**Areas（领域）- 需要持续维护的标准**
- 需要长期持续关注
- 维持在某个健康水平
- 永远不会"完成"
- 可以派生出多个项目
- 示例：健康管理、职业发展、财务规划、人际关系

**Resources（资源）- 外部参考材料**
- 来自外部的知识和信息
- 不是自己创造的内容
- 作为学习和参考的素材
- 示例：《卡片笔记写作法》阅读笔记、Docker 官方文档摘要、技术教程

**Archives（归档）- 不再活跃的内容**
- 曾经活跃但现在完成/暂停/取消
- 保留以便未来参考
- 不会频繁访问
- 按时间组织（年份）
- 示例：已完成的项目、已取消的项目、不再关注的领域

### 步骤 2：使用决策树快速分类

**PARA 分类决策树**：

```
┌─────────────────────────────────────┐
│    收到新的信息/笔记/文件           │
└──────────────┬──────────────────────┘
               ↓
        【问题 1】
   这是外部来源的参考资料吗？
   （书籍、文章、教程、文档等）
               │
       ┌───────┴───────┐
       │               │
      Yes             No
       ↓               ↓
   04-资源/        【问题 2】
   (Resources)   这个内容有明确的
                 "完成"状态吗？
                       │
               ┌───────┴───────┐
               │               │
              Yes             No
               ↓               ↓
          【问题 3】        【问题 4】
      有明确的截止    这需要长期持续
      日期或时间范围吗？  关注和维护吗？
               │               │
       ┌───────┴───────┐       │
       │               │   ┌───┴───┐
      Yes             No  Yes    No
       ↓               ↓   ↓      ↓
   02-项目/       【问题 5】 03-   10-卡片盒/
   (Projects)   这是已完成 领域/  或临时记录
                或停止的内容？Areas  (日志/收件箱)
                       │
               ┌───────┴───────┐
               │               │
              Yes             No
               ↓               ↓
           05-归档/        重新判断
          (Archives)      (返回问题1)
```

**决策流程文字版**：

1. **这是外部来源的参考资料吗？**
   - Yes → `04-资源/` (Resources)
   - No → 继续问题 2

2. **这个内容有明确的"完成"状态吗？**
   - No → 继续问题 4（可能是领域）
   - Yes → 继续问题 3（可能是项目）

3. **有明确的截止日期或时间范围吗？**
   - Yes → `02-项目/` (Projects)
   - No → 继续问题 5（检查是否归档）

4. **这需要长期持续关注和维护吗？**
   - Yes → `03-领域/` (Areas)
   - No → 检查是否是卡片盒笔记或临时记录

5. **这是已完成或停止的内容吗？**
   - Yes → `05-归档/` (Archives)
   - No → 重新判断（返回问题 1）

### 步骤 3：处理常见的边界模糊情况

**项目 vs 领域（最常混淆）**

关键判断：问自己"这会结束吗？"

如果 3 个月后还在做同样的事属于**正常**，那是**领域**。
如果 3 个月后还在做同样的事属于**失败**，那是**项目**。

示例：
- ❌ 错误："学习英语"放在项目（太宽泛，无终点）
- ✅ 正确：
  - **领域**：`03-领域/语言学习/`（持续关注）
  - **项目**：`02-项目/学习项目/3个月英语口语突破计划/`（有终点）

**资源 vs 卡片盒（知识来源 vs 个人洞察）**

关键判断：问自己"这是谁创造的？"

- **资源**：外部知识，保持原样，作为参考
- **卡片盒**：内化知识，用自己的话，独立洞察

转化关系：
```
阅读资料（Resources）
    ↓ 提炼
原子化卡片（Zettelkasten）
    ↓ 组合
输出文章（Output）
```

**项目 vs 归档（进行中 vs 已完成）**

归档时机：
- ✅ 项目状态标记为"✅ 已完成"或"❌ 已取消"
- ✅ 项目完成后 1 个月内未再访问
- ✅ 项目资料已提炼到卡片盒
- ❌ 项目暂停（⏸️ 暂停）：不归档，可能重启

### 步骤 4：定期重分类和优化

**每周回顾时检查**（5 分钟）：
- 检查进行中的项目：是否有项目实际上是领域？是否有项目已完成但未归档？
- 检查领域健康度：是否有领域需要派生项目？

**每月复盘时优化**（15 分钟）：
- 本月完成的项目是否已归档？
- 3 个月未查看的资源考虑删除
- 已提炼到卡片盒的资源可以精简

**常见重分类模式**：

- **项目 → 领域**：项目做了 6 个月仍未"完成"，目标设定不当，实际上是持续性工作
- **领域 → 项目**：领域下有明确的短期目标，拆分出可执行的项目推动领域发展
- **项目 → 归档**：项目状态标记为 ✅ 或 ❌，移动到 `05-归档/YYYY/`
- **资源 → 卡片盒**：深度阅读资源，产生个人理解，提炼核心概念/方法到卡片盒
- **收件箱 → PARA**：每日整理收件箱，根据决策树分类到对应位置

## 注意事项

- ⚠️ 不要追求完美分类，分类只是手段不是目的
- ⚠️ 当犹豫不决时，先放收件箱，每日整理时再决定
- ⚠️ 项目和领域的边界不是绝对的，可以灵活调整
- ⚠️ 避免创建过多的子文件夹，保持结构扁平
- ⚠️ 定期重分类是正常的，不要害怕调整
- ⚠️ 归档不等于删除，归档的内容仍然可以搜索和引用
- ⚠️ 资源笔记和卡片盒笔记要明确区分，避免混淆
