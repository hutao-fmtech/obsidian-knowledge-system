# 📦 归档

> 存放已完成的项目和不再活跃的内容

---

## 💡 什么是归档？

归档是 PARA 系统中的第四层（Archives），用于存放：
- ✅ 已完成的项目
- 🔚 不再活跃的领域
- 📚 过时的资源
- 📝 历史记录

### 归档 ≠ 删除

归档的内容：
- 不常用，但可能需要查看
- 有历史参考价值
- 完成后的经验总结
- 保留完整记录

---

## 🎯 归档原则

### 什么时候归档？

#### 项目归档
- ✅ 项目目标已达成
- ❌ 项目已取消或停止
- ⏸️ 项目长期暂停（6个月+）

#### 领域归档
- 不再是关注重点
- 生活阶段转变（如毕业、换工作）
- 已被其他领域替代

#### 资源归档
- 内容已过时
- 不再相关
- 很久没有查看

### 归档前检查

在归档前，先做这些：

```markdown
- [ ] 项目是否真的完成/停止？
- [ ] 有没有需要提炼的经验？
- [ ] 相关文档是否需要转存？
- [ ] 是否需要写复盘总结？
```

## 📋 归档流程

### 项目归档流程

1. **项目复盘**
   - 目标达成情况
   - 遇到的问题
   - 经验教训
   - 数据统计

2. **提炼知识**
   - 创建方法卡片 → [[10-卡片盒/方法卡片/README]]
   - 记录观点思考 → [[10-卡片盒/观点卡片/README]]
   - 更新相关MOC → [[10-卡片盒/索引卡片/README]]

3. **整理文档**
   - 删除临时文件
   - 整理关键文档
   - 补充必要说明

4. **移动归档**
   - 移动到归档文件夹
   - 更新状态为"已归档"
   - 添加归档日期

### 归档命名规范

```
YYYY-MM-项目名称/
YYYY-领域名称/
YYYY-MM-资源主题/
```

**示例**：
- `2025-12-用户系统重构/`
- `2025-学生时代/`
- `2025-Q4-技术学习资料/`

## 🔍 归档内容查找

### 如何找到归档内容？

#### 方法1：全局搜索
使用 Obsidian 搜索功能：
```
path:05-归档 关键词
```

#### 方法2：按日期浏览
根据归档时间定位文件夹

#### 方法3：查看归档索引
维护归档清单：

```dataview
TABLE file.folder as "位置", archived_date as "归档日期", type as "类型"
FROM "05-归档"
WHERE file.name != "README"
SORT archived_date DESC
LIMIT 20
```

## 💡 归档最佳实践

### 1. 及时归档
项目完成后及时归档，不要堆积

### 2. 记录元信息
在归档时添加：
- 归档日期
- 归档原因
- 关键成果
- 相关链接

### 3. 提炼经验
不要直接归档，先提炼：
- 方法论 → 方法卡片
- 经验教训 → 观点卡片
- 知识点 → 概念卡片

### 4. 保持整洁
- 删除临时文件
- 压缩大文件
- 整理文件结构

### 5. 定期清理
- 每年检查一次
- 删除确实无用的内容
- 重要内容做好备份

---

## ⚠️ 注意事项

### 不要过度归档
- ❌ 项目刚暂停就归档（可能会重启）
- ❌ 还在参考的资料
- ❌ 可能复用的内容

### 归档不是垃圾桶
- 真正无用的内容应该删除
- 归档的内容应该有保留价值
- 定期清理真正的垃圾

### 保护重要数据
- 重要项目归档前做好备份
- 敏感信息脱敏处理
- 大文件考虑外部存储
