# 📖 Life Knowledge System 使用手册

> 欢迎来到你的个人知识管理系统！这份手册将帮助你快速上手并充分利用这个系统。

---

## 🎯 系统概述

Life Knowledge System 是一个基于 **PARA + MOC + Zettelkasten** 混合架构的个人知识管理系统，帮助你：

- 📥 **高效收集**: 随时随地记录想法和信息
- 🗂️ **系统整理**: 自动分类和组织知识
- 🔗 **建立联系**: 连接相关知识，形成知识网络
- ✍️ **持续输出**: 基于积累创作有价值的内容

## 📂 文件夹结构说明

### 核心文件夹 (00-05) - PARA 架构

**00-系统** - 系统配置与导航
- 存放模板、配置文件和系统文档
- 主控制台 `🏠 HOME.md`
- 知识地图 `🗺️ 知识地图.canvas`

**01-收件箱** - 临时收集区
- `快速笔记/`: 临时想法，每日清理
- `待处理/`: 保存的资料，等待整理
- `灵感闪念/`: 创意点子，定期回顾

**02-项目** - Projects (有明确目标和截止日期)
- `工作项目/`: 职场相关项目
- `个人项目/`: 个人目标项目
- `学习项目/`: 系统学习计划

**03-领域** - Areas (持续关注，无截止日期)
- `职业发展/`: 技能、职业规划
- `技术领导力/`: 技术管理、团队带领
- `软件工程/`: 工程实践、架构设计
- `计算机从业/`: CS 基础理论
- `健康管理/`: 运动、饮食、睡眠
- `财务规划/`: 收支、投资、理财
- `人际关系/`: 家人、朋友、社交
- `兴趣爱好/`: 个人兴趣发展
- `读书写作/`: 阅读习惯、写作练习
- `公司知识管理/`（可选）: 团队知识体系建设，适合有团队知识管理需求时创建

**04-资源** - Resources (参考材料)
- `阅读笔记/`: 书籍读书笔记
- `课程学习/`: 在线课程笔记
- `工具/`: 工具调研与使用笔记
- `工具清单/`: 软件工具精选列表
- `媒体库/`: 视频、播客、影视收藏
- `参考资料/`: 文章、教程等外部资料

**05-归档** - Archive
- 已完成的项目
- 不再活跃的领域
- 按年份组织

### 专用文件夹 (10-90)

**10-卡片盒** - Zettelkasten 永久笔记
- `概念卡片/`: 定义核心概念
- `方法卡片/`: 记录可复用方法
- `观点卡片/`: 沉淀个人洞察
- `索引卡片/`: MOC 知识地图

**20-日志** - 时间线记录
- `日记/`: 每日记录 (YYYY/MM/YYYY-MM-DD.md)
- `周记/`: 每周回顾
- `月记/`: 每月复盘
- `年记/`: 年度总结

**30-输出** - 创作成果
- `微信公众号/`: 公众号文章（含 HTML 版本和配图）
- `博客文章/`: 博客和技术文章
- `技术分享/`: 演讲和团队分享
- `演讲稿/`: 演讲内容
- `作品集/`: 开源项目、工具等

**90-附件** - 媒体文件
- `images/cover-image/`: 文章封面图
- `images/illustrations/`: 文章配图
- `images/prompts/`: 图片生成提示词存档
- `files/`: 文档
- `audio/`: 音频

## 🔄 核心工作流

### 阶段1: 收集 (随时)

**何时收集？**
- 阅读时的想法
- 会议中的要点
- 突然的灵感
- 看到的有价值信息

**如何收集？**
1. 快速记录到 `01-收件箱/快速笔记/`
2. 保存外部资料到 `待处理/`
3. 记录创意到 `灵感闪念/`

**原则**: 不打断当前工作，快速记录即可

### 阶段2: 整理 (每日晚上15分钟)

**整理流程**:
1. 打开收件箱，逐条处理
2. 判断每条信息的类型:
   - 🎯 **可执行任务** → `02-项目/`
   - 🌟 **持续关注主题** → `03-领域/`
   - 📚 **参考资料** → `04-资源/`
   - 💡 **知识点/洞察** → `10-卡片盒/`
   - 🗑️ **无用信息** → 删除

3. 移动到相应文件夹
4. 清空收件箱

**判断标准**:
- 有截止日期 → 项目
- 需要长期关注 → 领域
- 仅供参考 → 资源
- 有独立价值的想法 → 卡片盒

### 阶段3: 链接 (每周回顾时)

**建立连接**:
1. 为新笔记添加双向链接 `[[相关笔记]]`
2. 更新相关的 MOC 索引
3. 使用标签分类 `#工作/项目` `#学习/阅读`
4. 检查孤岛笔记(无链接的笔记)

**链接类型**:
- 上游链接: 这个想法从哪里来
- 下游链接: 这个想法延伸出什么
- 横向链接: 相关的其他想法

### 阶段4: 输出 (根据创作计划)

**从卡片到文章**:
1. 选择输出主题
2. 收集相关卡片
3. 使用 MOC 作为大纲
4. 组合卡片成文章
5. 发布到 `30-输出/`
6. 反向链接到原始素材

## 📅 日常节奏

### 每日

**早晨 (06:00)**
- 打开 [[🏠 HOME]]
- 查看今天的日记
- 设定3个今日目标

**中午 (12:00)**
- 快速回顾上午工作
- 调整下午计划

**晚上 (21:00)**
- 整理收件箱 (15分钟)
- 更新工作日志
- 记录今日亮点
- 回顾目标完成情况

### 每周

**周一早晨**
- 使用周记模板创建本周笔记
- 设定本周核心目标
- 安排重要事项

**周五晚上**
- 周度回顾 (30分钟)
- 总结本周成就
- 反思需要改进的地方
- 规划下周重点

**周末**
- 整理卡片盒
- 更新 MOC 索引
- 阅读和学习
- 培养兴趣爱好

### 每月

**月初第一天**
- 设定月度 OKR
- 规划重点项目
- 制定学习计划

**月中检查**
- 检查各领域健康度
- 调整月度计划
- 处理偏离目标的情况

**月末最后一天**
- 月度复盘 (1小时)
- 数据统计
- 财务回顾
- 知识沉淀统计
- 下月规划

## 🤖 AI 自动化（OpenClaw 集成）

本系统支持通过 [OpenClaw](https://openclaw.ai) AI Agent 深度集成，让知识管理从「手动整理」升级为「AI 协作」。

### 功能概览

| 功能 | 如何使用 | 效果 |
|------|---------|------|
| **收集素材** | 直接发消息给 AI："帮我调研 XX 主题" | 研究结果自动保存到 `01-收件箱/待处理/` |
| **定时提醒** | 一次性设置，自动运行 | 每日收件箱整理、周回顾、月复盘定时提醒 |
| **内容整理** | 告诉 AI 整理方向 | 自动将笔记移动到正确文件夹 |
| **文章生成** | 提供大纲或素材 | AI 起草文章，可直接发布微信公众号 |
| **图片配图** | 让 AI 为文章配图 | 自动生成封面图 + 配图，存入 `90-附件/` |

### 快速配置

1. 安装 OpenClaw（详见 `CLAUDE.md`）
2. 设置定时任务：发送消息「结合我的 Obsidian 系统，帮我设置定时提醒」
3. AI 会自动创建三个 cron 任务：
   - **每日 21:00** → 收件箱整理提醒
   - **每周五 21:00** → 周回顾提醒
   - **每月 1 日 20:00** → 月复盘提醒

### AI 工作流示例

```
你 → AI："帮我调研 Zettelkasten 在工程师中的应用"
AI → 搜索、整理、写成笔记
AI → 自动保存到 01-收件箱/待处理/
你 → 晚上整理收件箱，提炼成卡片盒笔记
```

配置文件：`CLAUDE.md`（Claude Code）/ `AGENTS.md`（OpenClaw）/ `GEMINI.md`（Gemini）

## 💡 最佳实践

### 1. 保持收件箱清空

- 每天必须清理收件箱
- 不让信息堆积
- 养成整理习惯

### 2. 坚持每日记录

- 日记是系统的基石
- 简单记录即可
- 贵在坚持

### 3. 定期回顾

- 周回顾不可跳过
- 月复盘深度思考
- 回顾带来成长

### 4. 建立链接

- 新笔记至少2个链接
- 定期检查孤岛笔记
- 链接产生价值

### 5. 持续输出

- 每月至少1篇输出

- 不过度分类
- 不追求完美

## ❓ 常见问题

### Q: 收件箱的笔记太多整理不过来？

A: 说明你收集太多或整理太少。建议:
- 提高收集质量，只记录真正有价值的
- 每天必须清理，不能堆积
- 可以每天分2次整理，午间+晚间

### Q: 不知道该放项目还是领域？

A: 判断标准:
- 有明确截止日期 → 项目
- 需要长期关注 → 领域
- 例如: "完成XX课程"是项目，"持续学习"是领域

### Q: 卡片盒笔记怎么写？

A: 关键原则:
- 用自己的话转述
- 一卡一个概念
- 添加个人思考
- 建立至少2个链接

### Q: 标签和文件夹如何选择？

A:
- 文件夹: 物理位置 (在哪里)
- 标签: 内容属性 (是什么)
- 优先用文件夹组织，标签辅助分类

### Q: 多久能看到效果？

A:
- 1周: 养成记录习惯
- 1月: 形成工作流
- 3月: 知识开始关联
- 6月: 创作能力提升
- 1年: 建立知识体系
