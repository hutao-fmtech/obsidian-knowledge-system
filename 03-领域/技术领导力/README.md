# 🎯 技术领导力

## 📖 领域定义

**这个领域是什么？**
技术领导力（Technical Leadership）是作为 Tech Lead 的核心职责领域，涵盖技术决策、架构治理、团队管理、人才培养以及业务影响力。

**为什么重要？**
- 确保技术决策的质量与长远一致性。
- 提升团队整体效能，培养下一代技术人才。
- 在业务目标与技术债之间取得动态平衡。

**我的期望是什么？**
- 建立一套可复制的技术治理框架。
- 打造一支具备自驱力和高技术水准的团队。
- 提升个人的战略思维与跨部门协作影响力。

## 🎯 长期目标

### 1年目标
- 完善团队 1on1 机制，建立明确的成长路径。
- 落地 ADR (Architecture Decision Records) 机制。

### 3年目标
- 形成跨部门的技术标准领导力。
- 培养出至少 2 名能独立负责复杂业务线的 Tech Lead。

## 📅 日常习惯

### 每日习惯
- [ ] 巡视核心业务监控与错误日志。
- [ ] 关键 PR 的架构审查。

### 每周习惯
- [ ] 技术雷达更新与趋势同步。
- [ ] 核心成员深度 1on1。

### 每月习惯
- [ ] 团队产出与效能复盘。
- [ ] 技术深度分享或研讨。
